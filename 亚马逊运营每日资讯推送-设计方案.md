# 跨境电商·亚马逊运营「每日资讯推送」设计方案

> 版本：v1.0 ｜ 设计日期：2026-07-08 ｜ 适用对象：亚马逊中国卖家 / 运营团队 / 跨境品牌负责人
>
> 一句话目标：**用最低的阅读成本，把"今天发生了什么、对我的店铺有什么影响、我该立刻做什么"三条信息，每天稳定送到运营手上。**

---

## 一、产品定位

| 维度 | 说明 |
|------|------|
| 产品形态 | 每日结构化资讯卡片（公众号 / 企业微信 / 飞书 / 邮件 / Notion 均可承载） |
| 推送频率 | **工作日每日 1 期**（周一至周五 09:00 前送达；周末与法定节假日休刊或发「周报版」） |
| 单期体量 | 6–8 条精选条目，阅读时长 ≤ 5 分钟 |
| 内容标准 | 只收「对运营动作有影响的」信息：平台政策、费用、合规、流量、物流、工具更新 |
| 不收内容 | 软文、成功学、与卖家无直接关系的资本市场八卦 |

**核心设计原则**：每条资讯必须落到三层——**发生了什么 / 对我有什么影响 / 我今天要做什么**。缺任何一层，不入选。

---

## 二、资讯栏目体系（固定 7 个栏目）

每日版按固定顺序编排，便于读者形成阅读肌肉记忆：

| 序号 | 栏目 | 内容范围 | 优先级 |
|------|------|----------|--------|
| 1 | 🔝 今日头条 | 当天最重要的一条平台/政策变动，单独置顶 | 必选 |
| 2 | 📜 政策与费用 | 亚马逊官方规则、FBA 费用、佣金、账户健康 | 必选 |
| 3 | 🚀 流量与活动 | Prime Day、大促、广告、搜索算法、站内流量 | 必选 |
| 4 | 📦 物流与供应链 | FBA、海外仓、关税、运价、清关 | 必选 |
| 5 | 🌐 全球合规 | 税务、认证、AI 广告法、数据隐私、各国新政 | 必选 |
| 6 | 🛠 工具与运营 | Seller Assistant、AI Canvas、新工具、SOP | 选填 |
| 7 | ✅ 今日行动清单 | 从本期资讯提炼的 3–5 条可立即执行动作 | 必选 |

> 另设两个轻量模块：**「数据看板」**（可选，放运价指数/汇率/类目趋势图）与**「风险提示」**（红标紧急项，出现即插入头条下方）。

## 三、信息来源清单（采集层）

| 来源类型 | 具体渠道 | 采集方式 |
|----------|----------|----------|
| 官方一手 | Amazon Seller Central 公告、aboutamazon.com/news | RSS / 官网监测 |
| 国内媒体 | 雨果跨境、亿邦动力、网经社、AMZ123、知无不言 | 站点订阅 + 关键词抓取 |
| 海外媒体 | Marketplace Pulse、My Amazon Guy、Freightos、About Amazon | RSS / API |
| 政策源头 | 欧盟委员会公报、USTR、纽约州议会、各海关官网 | 政策 RSS |
| 运价/数据 | Freightos Baltic Index、Xeneta、各国汇率 | API 定时拉取 |

> 采集建议：用 RSS 聚合 + 关键词过滤（"Amazon / FBA / 关税 / 合规 / Prime"）+ 人工日清。自动化脚本见附带的 `daily_push_generator.py`。

## 四、推送节奏与时间线

| 时间 | 动作 |
|------|------|
| 08:00 | 自动化脚本拉取各源最新条目 |
| 08:30 | 编辑人工筛选、降重、补全「影响+动作」两层 |
| 09:00 | 发布到网页（GitHub Pages）/ 发送 QQ 邮件 / 微信通知 |
| 每周五 | 叠加「本周复盘」+ 下周大促日历 |
| 每月末 | 生成「月度政策汇总」长图/PDF |

## 五、内容模板（每条资讯的固定结构）

```
【标题】（一句话，含站点/地区/生效时间）
【标签】#栏目 #影响等级(高/中/低) #地区
【发生了什么】2–3 句事实陈述（含时间、数字、出处）
【对卖家的影响】这条信息会改变什么（成本/流量/合规风险）
【建议动作】具体、可执行、可验收的 1–2 步
【来源】媒体名 + 链接 + 发布日期
```

**影响等级定义**：
- 🔴 高：7 天内必须处理（如新税法生效、封号风险）
- 🟡 中：本季度需规划（如费用调整、新站点开放）
- 🟢 低：了解即可（趋势类、工具类）

## 六、视觉与排版规范

- **主色**：亚马逊橙 `#FF9900` + 深空灰 `#232F3E`
- **卡片化**：每条资讯独立卡片，左侧色条标识影响等级
- **移动优先**：单列流式布局，PC 端最大宽度 720px
- **可截图分享**：顶部固定「日期 + 期数 + 一句话摘要」，便于转发
- **无图不读**：关键政策配一张官方截图或数据图

## 七、配套交付物

本方案附带两个可直接使用的文件：

1. **`亚马逊每日资讯·示例第1期.html`** —— 按上述模板排版的样板期次，含 2026-07-08 真实最新资讯，可直接在浏览器打开、截图或导入公众号。
2. **`daily_push_generator.py`** —— 读取结构化条目（JSON）自动渲染同款 HTML 的脚本，把"采集→模板"闭环跑通。

---

## 八、自动化实现思路（下一步）

1. **采集**：RSSHub + 官方公告页定时抓取，写入 `items.json`。
2. **过滤**：按关键词 + 影响等级打分，初筛候选条目。
3. **生成**：`daily_push_generator.py` 渲染 HTML 网页。
4. **分发**：GitHub Pages 部署网页 / QQ 邮件 SMTP / PushPlus 微信通知。
5. **归档**：每日文件按日期落盘，月底汇总。

> 想要真正"每日自动推送"，只需把脚本挂到 cron（工作日 08:00），并把采集源接上 RSS/API。本方案已把模板和生成器准备好，落地成本极低。
