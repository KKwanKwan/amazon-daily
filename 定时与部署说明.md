# 定时与部署说明（每天北京时间 08:00 自动更新网页 + 往期归档）

## 你的使用方式

> **每天打开同一个网址，就是当天最新一期；想看历史就点页脚的「📚 查看往期归档」。**

部署完成后你会得到一个固定网址（GitHub Pages），例如 `https://你的用户名.github.io/仓库名/`：
- 每天 **北京时间 08:00** 自动重新生成并发布，刷新即见新内容；
- 每期页脚有「📚 查看往期归档」入口，进入后有 **本周复盘 / 月度汇总 / 完整往期列表**；
- 所有往期都持久化在仓库的 `archive/history.json`，并据此重生成各往期网页，不会丢失。

完整链路：`collector.py` 采集 → `run_daily.py` 写入档案并生成网页 → GitHub Actions 每天 08:00 跑、提交档案、部署 Pages。

---

## 一键部署（推荐：GitHub Actions + Pages）

1. **建仓库**：GitHub 新建仓库（如 `amazon-daily`），把本目录全部文件推上去。
2. **开 Pages**：仓库 **Settings → Pages → Build and deployment → Source 选 "GitHub Actions"**。
3. **（可选）配通知**：默认只出网页，不发邮件/微信。想要提醒时再配 Secrets 并用 `--notify` 运行（见文末）。
4. **完成**：工作流 `cron: '0 0 * * *'`（UTC 00:00 = 北京时间 08:00）每天自动执行；每次会把 `archive/history.json` 提交回仓库以保证往期持久化。
5. **立刻验证**：仓库 **Actions** 页面对应工作流点 **Run workflow** 手动跑一次，看 Pages 是否出页、archive 是否生成。

> 网页固定地址在 Actions 运行日志 `deployment` 步骤的 `url` 里，或 Settings → Pages 查看。

## 本地预览 / 手动运行

```bash
pip install requests
python3 run_daily.py                 # 采集+生成网页（含往期归档），不发送
python3 run_daily.py --hours 72 --top 8
# 浏览器打开 site/index.html（最新）或 site/archive.html（归档）
python3 run_daily.py --notify        # 额外发邮件/微信（需先配 .env）
```

## 采集源说明（原文直链 + 安全）

- **官方 RSS（已实测可用，返回【原文直链】）**：About Amazon、Practical Ecommerce。
  想加你信任的媒体，编辑 `collector.py` 里的 `OFFICIAL_FEEDS` 追加即可。
- **Google News 聚合（广度补充，中文覆盖依赖它）**：链接为 Google News 聚合页（已标注
  `via Google News 聚合`），因其链接不重定向到原文，故作为补充而非主源。
- **去重策略**：同一标题若官方源与 Google News 同时出现，保留官方（原文直链）。
- **安全/可行**：所有请求带超时；单个源解析失败仅跳过不影响整体；输出全部转义；无命令注入。

## 往期归档（必须记录）

- 每日数据写入 `archive/history.json`（提交到仓库 = 持久化）。
- 每次运行都从 `history.json` 重生成：最新一期 `index.html`、各往期 `亚马逊每日资讯_日期.html`、归档页 `archive.html`。
- 即使某天工作流异常，历史数据仍在仓库里，恢复后重生成即可补齐。

---

## 本周复盘 / 月度汇总

- `site/archive.html` 自动包含：
  - **本周复盘**：最近 7 天各期条数 + 高影响数；
  - **月度汇总**：按月份统计条数；
  - **完整往期列表**：全部日期，点击直达当日期刊。

## （可选）邮件 / 微信通知

默认关闭。需要时：
1. `cp .env.example .env`，填入 QQ 邮箱(`SMTP_USER`/`SMTP_PASS`授权码/`MAIL_TO`) 或 `PUSHPLUS_TOKEN`（微信）。
2. 运行加 `--notify`：`python3 run_daily.py --notify`。
3. GitHub Actions 里则在工作流 `采集` 步骤加环境变量（Secrets）并把运行命令改为 `python3 run_daily.py --notify`。

QQ 邮箱授权码获取：邮箱设置 → 账户 → 开启 IMAP/SMTP 服务 → 发短信得授权码。

## 自有服务器部署（备选）

```bash
# cron（Asia/Shanghai）
(crontab -l 2>/dev/null; echo 'CRON_TZ=Asia/Shanghai'; \
 echo '0 8 * * * cd /path/to/workspace && /usr/bin/python3 run_daily.py >> daily.log 2>&1') | crontab -
```
网页托管：用 Nginx 指向 `site/` 目录并反代到公网域名。往期档案 `archive/` 随仓库提交即可持久化。
