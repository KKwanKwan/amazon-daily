# 零基础部署步骤图解（不用懂代码，照着点就行）

## 一句话先搞懂

把 GitHub 想成一个**免费的「网盘 + 定时机器人」**：
- **网盘**：帮你存这一套文件和生成的网页；
- **定时机器人**：每天北京时间 08:00 自动运行，帮你生成当天网页。

你今天只做三件事：**建网盘 → 传文件夹 → 开一个开关**。之后每天打开一个网址就能看，什么都不用管。

> ⚠️ 你**不需要读、不需要改、不需要复制**任何代码。文件夹里的一切我都已准备好。

---

## 第 0 步：先把文件弄到你自己电脑上

左边/本地的 `workspace` 文件夹里，就是我给你生成好的全部文件。
- 把这整个文件夹**下载 / 打包(zip)** 到你的电脑桌面，命名为 `amazon-daily`（名字随意）。
- 文件夹里至少包含这些（你不用管它们干嘛，传上去就行）：

```
amazon-daily/
├─ .github/workflows/daily_push.yml   ← 这就是「每天8点自动运行」的定时器（已在里面）
├─ collector.py
├─ run_daily.py
├─ daily_push_generator.py
├─ push.py
├─ archive/                           ← 往期记录（会自动累积）
├─ site/                             ← 生成的网页
├─ 亚马逊运营每日资讯推送-设计方案.md
├─ 定时与部署说明.md
└─ .env.example
```

## 第 1 步：注册 GitHub（像注册微信一样简单）

```
┌─────────────────────────────────────────────┐
│  打开浏览器，地址栏输入：github.com          │
│                                             │
│  点右上角绿色的【Sign up】按钮               │
│   → 输入邮箱、设密码、取个用户名             │
│   → 一路点【Continue / 验证】直到注册完成   │
└─────────────────────────────────────────────┘
```
📌 记好你的**邮箱 + 密码**（以后登录用）。

---

## 第 2 步：新建一个「网盘」（仓库）

```
┌─────────────────────────────────────────────┐
│  登录后，点页面右上角的 【+】（加号）          │
│   → 选 【New repository】（新建仓库）         │
│                                             │
│  Repository name（仓库名）：填  amazon-daily │
│  （这是你这个网盘的名字，随便起也行）        │
│                                             │
│  ⚠️ 务必选：【Public】（公开）               │
│  其它不用改，点绿色【Create repository】     │
└─────────────────────────────────────────────┘
```
📌 点完会跳到一个新页面，别慌，继续第 3 步。

---

## 第 3 步：把文件夹传上去（关键！就是「上传」）

```
┌─────────────────────────────────────────────┐
│  在新页面里找这一行字，点它：                │
│   "…or upload an existing file"              │
│   （中文大概叫：…或上传已有文件）            │
│                                             │
│  进入上传页后：                             │
│   👉 把桌面上的 amazon-daily 文件夹          │
│      直接【拖拽】到虚线框里；              │
│      或点【choose your files】选中所有文件  │
│                                             │
│  传完后，拉到页面最下面：                   │
│   点绿色大按钮【Commit changes】            │
│   （意思是「确认保存」）                     │
└─────────────────────────────────────────────┘
```
✅ 到这里，文件已经上网盘了。你**一个字都没改、没复制代码**。

---

## 第 4 步：打开「自动出网页」的开关（只此一处要设置）

```
┌─────────────────────────────────────────────┐
│  回到你的仓库页面，点上方菜单里的【Settings】│
│                                             │
│  在左侧栏找到并点 【Pages】                 │
│                                             │
│  找到 "Build and deployment"（构建与部署）  │
│   → Source（来源）改成：【GitHub Actions】  │
│   → 页面会自动保存                          │
└─────────────────────────────────────────────┘
```
📌 这个开关就是告诉 GitHub：「用我传上去的那个定时器来自动发布网页」。

---

## 第 5 步：立刻试一次（不用等明天）

```
┌─────────────────────────────────────────────┐
│  回到仓库页面，点上方菜单【Actions】         │
│                                             │
│  左边会看到工作流「亚马逊每日资讯推送」      │
│   → 点它 → 点右边【Run workflow】           │
│   → 再点绿色的【Run workflow】              │
│                                             │
│  等 1~2 分钟，出现绿色✅就成功了            │
└─────────────────────────────────────────────┘
```
📌 以后不用手动点，每天北京时间 08:00 它自己跑。

## 常见问题

**Q：我是不是要把代码复制到某个框里？**
A：不用。你传的是整个文件夹，`.github/workflows/daily_push.yml` 已经在里面，GitHub 会自动识别它当定时器。

**Q：要花钱吗？**
A：GitHub 公开仓库 + Pages 完全免费。

**Q：我传完要不要每天手动点？**
A：不用。定时器每天 08:00 自动跑、自动更新网页。你只负责「打开网址看」。

**Q：以后想加邮件/微信提醒怎么办？**
A：暂时不用管（你说过先只出网页）。以后想要，再来找我，我教你填一个 `.env` 并加一个开关即可。

**Q：如果我不想用 GitHub 呢？**
A：也可以把这程序装在你自己电脑上、让你电脑每天 8 点跑——但那样你的电脑那天必须开着。如果你更想要这个方案，告诉我，我给你做一版「双击就能跑」的电脑版。
